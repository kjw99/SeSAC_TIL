## 어제 내용 복습 + 클래스 생성 연습

- 기능, 변수 등을 하나로 모아서 객체로 만들어서 쓰자.
- 클래스 : 객체
- 클래스 이름은 파스칼케이스 활용. ex) SomeClassName
    - 대문자로 시작. 이어질 단어는 붙이고 첫 글자를 대문자로.
- 공통된 기능은 클래스를 통해 구현하고 상속 받아서 사용할 수 있다.
    - Person의 밥먹기 등..
- 가르치다. 라는 행위는 선생님 외에도 AI가 할 수도 있을 것.
    - abstractmethod 를 활용해서 가르치다! 라는 행위를 자식 클래스가 반드시 구현할 수 있도록 추상 메서드로 생성
    - 이걸 상속 받아서 같은 행위지만, 서로 다른 행동을 하도록 구현하는 것.
- 굳이 가르치다라는 행위를 분리할 필요가 없을 수도 있다?
    - Person 클래스 생성
    - role 변수를 만들고, 그 안에 선생, 학생 등 또 다른 클래스를 저장.
    - 이런 방식이 Composition 을 활용한 것.
    - 뭐가 더 유용한 지는 상황에 따라 다름!
- 스마트 홈을 예시로 생각해보기.
    - 목적 : 여러 기기들을 중앙에서 관리하고 싶다.
    - 기기(Device)
        - 켜기()
        - 끄기()
        - 상태 확인하기() - 오버라이딩
    - 에어컨(기기)
        - 온도 조절 기능
    - 조명(기기)
        - 밝기 조절 기능
    - 디스플레이(기기)
        - 채널 변경 기능
    - 스마트 허브 : Device 들 관리! 기기랑은 Composition으로 연결하면 좋을 듯
        - 연결된 기기 : list형태 / Composition 방식으로 기기와 연결되어 있다!
        - 연결하기()
        - 연결 방식이 무엇이니?
- 이렇게 많이 구현하다 보면 코드가 길어짐.
- 그래서 생긴 규칙 중 하나가 하나의 파일엔 하나의 클래스만 존재하자! 라는 규칙이 있다.
